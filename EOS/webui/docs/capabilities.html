Capabilities

Capabilities & Autonomy

EOS has four autonomy dimensions that gate what it can perceive, think, do, and initiate. All are live-adjustable from the admin panel.

Capabilities

Where to Find This

Navigate to Admin Panel → Control & Permissions → Capabilities. Changes are immediate — no restart required. Settings reset to config.json values when EOS restarts. To make a change permanent, edit config.json directly.

Capabilities

The Four Autonomy Dimensions

Each dimension controls a distinct class of behavior. They can be toggled independently.

Perception
Observing the environment — screen capture, sensor data, and passive input streams. Disabling this stops EOS from watching the environment between turns.
Cognition
Background thinking, reflection loops, and idle cognition cycles. Disabling this stops EOS from reasoning between turns but does not affect responses to your messages.
Action
Tool use and external integrations. Disabling this makes EOS text-only — it can still converse but cannot call tools, write files, or reach external services.
Initiative
Proactive, unsolicited actions — EOS reaching out or acting without being asked. Disabling this makes EOS fully reactive; it only acts in direct response to your messages.
Recommended Settings
For general use, all four on is fine. For a more passive assistant, turn off Action and Initiative. Cognition can be left on — it only affects background thinking, not actions.
Capabilities

Computer Use

Computer Use has its own enabled toggle and a separate Default Mode setting. The enabled toggle must be on for any computer use to occur. The mode determines how much autonomy EOS has when it is active.

Off
No computer use regardless of the enabled flag. EOS cannot interact with the desktop or launch applications.
Command Only
Approved applications only. EOS acts on explicit user requests and pauses for confirmation before each action. Safe for general use.
Supervised Session
Approved applications with bounded continuation while you watch. EOS can take a short sequence of actions before pausing for review. Use when you want to delegate a task but stay in the loop.

See the full Computer Use guide for the application allowlist, the HALT button, and confirmation queue details.

Workspace Permissions

Two additional toggles control what EOS can do inside the workspace directory. Both default to off.

  • Allow Delete — whether EOS can permanently delete files in the workspace. Off by default.
  • Allow Exec — whether EOS can run shell commands. Off by default.
Shell Execution
Allow Exec enables shell command execution inside the workspace. Only enable if you explicitly need it. With Allow Exec on, EOS can run arbitrary commands in response to instructions.
Capabilities

Creativity

The creativity subsystem injects lateral reframes and novel angles into EOS's thinking. Three controls govern it:

  • Enabled — master toggle for the creativity subsystem. Off means no creative reframes at all.
  • Frequency — how often creative reframes inject: Off, Low, Medium, or High.
  • Autonomous Idle — whether the creativity subsystem runs during idle cognition cycles when no conversation is active.
Frequency Tip
Start with Medium frequency. Low is subtle and easy to miss. High can feel intrusive — EOS will frequently reframe or offer unexpected angles even on straightforward requests.
Capabilities

Google

When Google Workspace is connected, Calendar, Gmail read, and Drive access are always enabled. Only outbound email sending requires an extra permission toggle:

  • Gmail Send — off by default. Enable only if you want EOS to send emails on your behalf. Marked as a sensitive permission.

If Google is not connected or the credential file is missing, all Google capabilities are unavailable regardless of these settings. See the Integrations page for setup details.

Capabilities

Making Changes Permanent

Admin panel changes reset on restart. To persist them, edit config.json. The relevant blocks are:

Autonomy Defaults

"autonomy_defaults": {
  "perception": true,
  "cognition": true,
  "action": true,
  "initiative": true
}

Computer Use

"computer_use": {
  "enabled": false,
  "default_mode": "command_only"
}

Workspace Tools

"workspace_tools": {
  "allow_delete": false,
  "allow_exec": false
}

Creativity

"creativity": {
  "enabled": true,
  "frequency": "medium",
  "autonomous_idle": false
}

Google

"google": {
  "gmail_send_enabled": false
}
Workflow Tip
The admin panel is great for experimenting. Once you find settings you like, copy them to config.json so they survive restarts.
