Discord

Connect EOS to your Discord server as a bot. It responds to mentions by default and supports a handful of slash commands.

Capabilities

What it Does

💬
Responds to Mentions
Tag @BotName in any channel EOS can read and it will reply in that channel, just like a conversation in the chat UI.
⚡
Slash Commands
Use !initiative to toggle autonomous initiative on or off, and !status to display a live capability map directly in Discord.
🗨️
Text Conversations
Full EOS conversation capability through Discord messages — the same memory, tools, and reasoning available in the chat UI.

Setup

  1. 1
    Create a Discord Bot
    Go to the Discord Developer Portal (discord.com/developers/applications), create a new application, then open the Bot tab and create a bot user. Copy the bot token — you will need it in the next step.
  2. 2
    Save the Token
    Create or open the file AI personal files\Discord.txt and paste the bot token as a single line with no extra whitespace. EOS reads credentials from this file at startup.
  3. 3
    Invite the Bot to Your Server
    In the Developer Portal, use the OAuth2 URL Generator to create an invite link. Grant the bot Send Messages and Read Message History permissions. In the Bot tab, enable the Message Content Intent — without this, the bot cannot read message text.
  4. 4
    Enable Discord in Config
    Confirm that config.json contains "discord": {"enabled": true}. If the block is missing, add it. See the Configuration section below for the full block.
  5. 5
    Restart EOS
    The Discord bot starts automatically when EOS boots and the enabled flag is true. After restarting, your bot should appear online in Discord within a few seconds.

Configuration

The full Discord block in config.json:

"discord": {
  "enabled": true,
  "credential_file": "AI personal files/Discord.txt",
  "respond_only_to_mentions": true,
  "ignore_bots": true
}
Field Description
enabled Master toggle. Set to false to disable the Discord bot entirely without removing the config block.
credential_file Path to the file containing the bot token. Relative to the EOS root directory. Default: AI personal files/Discord.txt.
respond_only_to_mentions When true, EOS only responds when directly mentioned with @BotName. Set to false to respond to all messages in channels it can read — use with caution in busy servers.
ignore_bots When true, EOS ignores messages from other bots. Recommended to leave on to avoid feedback loops.

Checking Status

Open the Admin Panel → Integrations tab. The Discord row shows:

  • Whether the credential file was found
  • Whether the bot is currently connected to Discord
  • Any connection errors with detail
First Stop for Bot Issues
If the bot is not responding, check the Integrations tab before anything else. A missing credential file and a bad token produce different errors — the tab distinguishes between them.

Troubleshooting

Symptom Likely Cause & Fix
Bot not appearing online in Discord The token in Discord.txt is incorrect or has extra whitespace. Open the file, verify the token is a single line with no spaces or line breaks, and restart EOS.
Bot is online but not responding to mentions Message Content Intent is not enabled in the Discord Developer Portal. Go to your application's Bot tab and enable it. This is required for the bot to read message text.
Bot responding to every message, not just mentions Set "respond_only_to_mentions": true in the discord block in config.json and restart EOS.
