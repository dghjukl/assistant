Overview

What is EOS?

A local AI companion and persistent non-human entity built to run entirely on your machine — no cloud required.

🏠
Fully Local
Runs entirely on your machine. No cloud API key required. Nothing leaves your machine — ever.
🧠
Persistent Identity
Maintains a consistent persona and long-term memory across every session.
🔒
Private by Design
Your conversations, files, and data stay on your hardware. Zero telemetry.
💻
Windows Native
Designed and optimised for Windows 10 and Windows 11. Runs without WSL or Docker.
Architecture

How It Works

EOS is a coordinator. It discovers what is running and builds a capability map on startup — missing helpers degrade gracefully with no hard failures.

  1. 1
    Start backend model servers
    Use a launcher script (e.g. Launch EOS.bat or launchers\start-standard.bat) to bring up the model servers. Each opens its own console window.
  2. 2
    Start EOS
    Run start-eos.bat. EOS initialises its memory system, loads configuration, and begins listening.
  3. 3
    EOS discovers services and builds a capability map
    On startup EOS probes each configured endpoint. The result is a live capability map — you can see it printed in the console and in the Admin Panel.
  4. 4
    Missing helpers degrade gracefully
    If a backend is not running, that capability is marked unavailable or degraded. EOS keeps working at reduced capability — no crash, no hard failure.
  5. 5
    Open the interface in your browser
    Navigate to http://127.0.0.1:7860/. The chat interface is ready. The Admin Panel is at http://127.0.0.1:7860/admin.
Before You Begin

System Requirements

EOS is designed to run on modest consumer hardware. A GPU is recommended but not required.

Component Minimum Recommended
Operating System Windows 10 64-bit Windows 11
Python 3.10 3.11
RAM 16 GB 32 GB
GPU Not required (CPU inference) NVIDIA with 8 GB+ VRAM
Disk Space 20 GB free 30 GB free
CUDA Not required CUDA 13.1 compatible driver
