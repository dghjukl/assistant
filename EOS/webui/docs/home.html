Overview

What is EOS?

EOS is the local platform for running a persistent AI entity on your machine. The entity is the live runtime instance; EOS is the thing you install, launch, and configure.

🏠
Fully Local
Runs entirely on your machine. No cloud API key required. Nothing leaves your machine.
🧠
Persistent Entity
The runtime intelligence maintains identity continuity and long-term memory across sessions.
🔒
Private by Design
Your conversations, files, and data stay on your hardware. Zero telemetry.
💻
Windows Native
Designed and optimized for Windows 10 and Windows 11. Runs without WSL or Docker.
Defaults

Canonical Entry Points

There are multiple scripts in the repo, but new users should treat these as the default hierarchy.

  1. 1
    Install with setup\Setup-Full.ps1
    This is the recommended install path for most users.
  2. 2
    Launch backends with launchers\start-standard.bat
    This is the recommended backend bundle for most users.
  3. 3
    Bootstrap EOS with start-eos.bat
    EOS discovers running services, builds the capability map, and opens the WebUI.
  4. 4
    Deviate only for a specific reason
    Use Lite install, minimal/full bundles, vision, or per-server launchers only when you know why you need them.
