Capabilities

Google Workspace

EOS can read your Google Calendar, search Gmail, and browse Google Drive. When enabled, it can also send email on your behalf.

Overview

What it Does

📅
Google Calendar
Read upcoming events and answer questions about your schedule.
✉️
Gmail
Search your inbox, read threads, and answer questions about emails.
📁
Google Drive
List and read files, find documents by name or content.
📤
Send Email
Compose and send via Gmail — disabled by default, requires explicit enable.
Configuration

Setup

  1. 1
    Create a Google Cloud project
    Create a new project in the Google Cloud Console and enable the Calendar, Gmail, and Drive APIs.
  2. 2
    Configure the OAuth consent screen
    Set the application type to Desktop App and the user type to External (personal use). You do not need to publish the app — leave it in testing mode and add your Google account as a test user.
  3. 3
    Download the client secret JSON
    In Google Cloud Console, go to APIs & Services → Credentials, create an OAuth 2.0 Client ID for a Desktop application, and download the JSON file.
  4. 4
    Place the file in AI personal files
    Copy the downloaded file into your AI personal files\ folder. The filename must match the pattern client_secret_*.json — do not rename it to anything else.
  5. 5
    Enable Google in config.json
    Add or update the following in config.json:
    "google": {
  "enabled": true
}
  6. 6
    Restart EOS
    On first run after enabling, EOS will open a browser window to complete OAuth authorization. Keep the browser window accessible.
  7. 7
    Authorize in your browser
    Follow the prompts to grant EOS access to your Google account. EOS saves the resulting token to data\google_token.json — future starts will not require re-authorization unless the token expires.

Verification

Checking Integration Status

Open the Admin Panel and navigate to the Integrations tab. The Google section shows:

  • Credential file found (yes / no)
  • OAuth token valid (yes / no)
  • Which services are enabled
  • Live data previews: upcoming calendar events, inbox preview, and recent Drive files

If any service shows as unavailable, the status panel will indicate whether the issue is a missing credential file, an expired token, or a disabled service flag.

💡
Token expired?
Delete data\google_token.json and restart EOS. It will open a new browser authorization window and save a fresh token.
Optional Feature

Enabling Gmail Send

Gmail Send is off by default. EOS can read and search your email without it — you only need to enable this if you want EOS to send messages on your behalf.

Enable at runtime

Admin Panel → Control & Permissions → Capabilities → Google → Gmail Send. This takes effect immediately and does not require a restart.

Make it permanent

To keep Gmail Send enabled across restarts, add the following to config.json:

"google": {
  "enabled": true,
  "gmail_send_enabled": true
}
⚠️
Understand what you're enabling
With Gmail Send enabled, EOS can send email on your behalf. It will always tell you what it's sending and why, but only enable this if you've established trust with your EOS instance.
Help

Troubleshooting

Symptom Fix
No credential file found Check that the file is in AI personal files\ and the filename matches client_secret_*.json.
OAuth browser window not opening Confirm "enabled": true is set under "google" in config.json.
Token expired / auth errors Delete data\google_token.json and restart EOS to re-authorize.
Calendar events not showing Check that "calendar_enabled": true is set in config.json under the Google block.
