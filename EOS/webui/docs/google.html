Configuration

Setup

  1. 1
    Create a Google Cloud project
    Create a new project in the Google Cloud Console and enable the Calendar, Gmail, and Drive APIs.
  2. 2
    Configure the OAuth consent screen
    Set the application type to Desktop App and the user type to External (personal use). You do not need to publish the app — leave it in testing mode and add your Google account as a test user.
  3. 3
    Download the client secret JSON
    In Google Cloud Console, go to APIs & Services → Credentials, create an OAuth 2.0 Client ID for a Desktop application, and download the JSON file.
  4. 4
    Place the file in AI personal files
    Copy the downloaded file into your AI personal files\ folder. The filename must match the pattern client_secret_*.json — do not rename it to anything else.
  5. 5
    Enable Google in config.json
    Add or update the following in config.json:
    "google": {
  "enabled": true
}
  6. 6
    Restart EOS
    Restart EOS so the WebUI loads the Google integration and detects your client secret file.
  7. 7
    Open the Admin Panel and click Connect Google Account
    Go to Admin Panel → Integrations, click Connect Google Account, and complete the browser prompts. EOS saves the resulting token to data\google_token.json. Already-authorized users are not interrupted on startup; you only need this flow the first time or when re-authorizing.

Verification

Checking Integration Status

Open the Admin Panel and navigate to the Integrations tab. The Google section shows:

  • Credential file found (yes / no)
  • OAuth token valid (yes / no)
  • Which services are enabled
  • Live data previews: upcoming calendar events, inbox preview, and recent Drive files

If any service shows as unavailable, the status panel will indicate whether the issue is a missing credential file, an expired token, or a disabled service flag.

💡
Token expired?
Delete data\google_token.json, restart EOS, then use Admin Panel → Integrations → Re-authorize to save a fresh token.
Help

Troubleshooting

Symptom Fix
No credential file found Check that the file is in AI personal files\ and the filename matches client_secret_*.json.
OAuth browser window not opening Open Admin Panel → Integrations and click Connect Google Account. If nothing opens, confirm "enabled": true is set under "google" in config.json and allow pop-ups for the WebUI.
Token expired / auth errors Delete data\google_token.json, restart EOS, then use Admin Panel → Integrations → Re-authorize.
Calendar events not showing Check that "calendar_enabled": true is set in config.json under the Google block.
