Quick Start

Using EOS

How to start, stop, and interact with EOS day-to-day.

Startup

Starting EOS

EOS starts in two steps: first bring up the backend model servers, then start the EOS interface. Always in that order.

Step 1 — Start a backend launcher

Choose the profile that matches what you need. More backends mean more capability but use more RAM and VRAM.

Profile Script What starts Use when
Minimal start-minimal.bat Main model + tool helper Low RAM, basic tasks only
Standard start-standard.bat Main + tool helper + thinking helper Recommended for everyday use
Full start-full.bat Standard + creativity model Creative writing, brainstorming
Add Vision Your bundle + start-vision-gpu.bat Any profile above + vision server Image understanding (GPU required)

Or use the GUI launcher: double-click Launch EOS.bat, select a profile, and click Start. The launcher manages all backend windows for you and shows live status for each server.

Step 2 — Start the EOS interface

Once each backend window has printed its ready message, run:

start-eos.bat

Wait for the console to print:

Starting WebUI at http://127.0.0.1:7860/

Then open your browser to that address. EOS is ready.

Interface

The Chat Interface

The main interface lives at http://127.0.0.1:7860/. It is a standard chat UI with a few EOS-specific features.

Text chat

Type your message in the input box and press Enter or click Send. EOS will respond in the chat window. There is no session limit — conversations can be as long as you need.

ⓘ
Persistent memory
EOS does not reset between sessions. It remembers previous conversations, your preferences, and context from past interactions. Closing the browser or restarting EOS does not wipe memory — everything is persisted to disk.

Voice input (STT)

When speech-to-text is active, a microphone button appears in the input area. Hold the button to speak, then release to transcribe and send. Voice input requires the STT service to be running — if the microphone button is absent, STT is not active.

Voice output (TTS)

When text-to-speech is active, EOS speaks its responses automatically using the Piper TTS engine. Audio plays through your default output device. TTS can be toggled in the Admin Panel without restarting EOS.

Shutdown

Stopping EOS

EOS can be stopped at any time without data loss. All memory and state are written to disk continuously.

  1. 1
    Stop the EOS interface
    Press Ctrl+C in the start-eos.bat console window, or simply close that window. The WebUI will become unavailable immediately.
  2. 2
    Stop the backend servers
    Close the backend launcher windows (or click Stop in the GUI launcher). Model servers shut down and release RAM and VRAM. You can leave them running between EOS restarts if you prefer faster startup next time.
💡
Nothing is lost on shutdown
Memory, conversation history, and all state are persisted to disk. The next time you start EOS it will resume exactly where it left off.
Diagnostics

Checking System Status

Two ways to see what is running and what is not.

Script

status-eos.bat

Or from a terminal:

python eos.py --status

Both commands probe each configured endpoint and print the current capability map — which services are active, which are degraded, and which are unavailable.

Status meanings

active
Service is running and responding normally. Full capability available.
degraded
Service is reachable but running in a reduced mode, or a fallback is in use. Capability is partial.
unavailable
Service is not running or not reachable. That capability is cleanly disabled — no errors, just absent.

Admin Panel

The Admin Panel at http://127.0.0.1:7860/admin shows a live view of the capability map with the same status indicators, plus per-service response time and error counts.

Resilience

Capability Degradation

EOS is designed to keep working when backends are missing. The table below shows what happens when each optional service is absent.

Missing service Effect on EOS
Tool helper (port 8082) Tool calls fall back to the main model. Slightly slower and less reliable for complex multi-step tool use.
Thinking helper (port 8083) Extended reasoning falls back to the main model. Step-by-step analysis still works, just without a dedicated reasoning pass.
Vision server Vision capability is cleanly unavailable. EOS will tell you it cannot process images. All other functions unaffected.
STT service Voice input unavailable. Microphone button does not appear. Text chat is fully unaffected.
TTS service Voice output unavailable. Responses appear as text only. Text chat is fully unaffected.
Discord credentials Discord integration silently skipped on startup. No error — bot simply does not connect.
Google OAuth credentials Calendar, Gmail, and Drive integrations silently skipped. All other capabilities unaffected.
💡
Run the minimum for your task
You do not need all backends running at the same time. For a quick question, start-minimal.bat (main + tools) is enough. Add thinking or vision only when you need them. This saves RAM and lets backends start faster.
