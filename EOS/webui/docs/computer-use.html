Capabilities

Computer Use

EOS can launch and interact with approved desktop applications. This is a layered safety system — you define which apps remain entrusted and what EOS can do in them.

!
Safety first
Computer Use is powerful. The admin controls exist so you can move EOS into a narrower or safer posture immediately whenever trust, context, or risk changes.
Modes

Operating Modes

Computer Use has three modes. The active mode determines how much supervision and continuation EOS has when acting through the desktop.

Off
No computer use, regardless of any other setting. EOS will not launch or interact with any application.
Command Only
Approved apps only. EOS takes single actions in direct response to explicit user-requested tasks and pauses after each step.
Supervised Session
Approved apps with bounded continuation. EOS can take multiple steps toward a goal while you're watching, but stays within the scope you define.
💡
Recommended starting point
Command Only is the easiest way to narrow EOS into a safer, more supervised posture when you want hands-on oversight.
Configuration

Configuring Computer Use

  1. 1
    Set the runtime posture in the Admin Panel
    Admin Panel → Control & Permissions → Capabilities → Computer Use → set Enabled and choose the supervision mode that matches the current level of trust and risk.
  2. 2
    Set the persisted default mode
    Set Default Mode to Command Only or Supervised Session in the Admin Panel. Alternatively, edit config.json directly:
    "computer_use": {
  "enabled": true,
  "default_mode": "command_only"
}

    Use "supervised_session" for the Supervised Session mode.

  3. 3
    Narrow or revoke at any time
    The active mode can be changed from the Admin Panel at any time without restarting EOS. Move to a narrower mode or to Off whenever you need a safer posture.
App Allowlist

Approving Applications

EOS will only interact with applications that appear on the allowlist. The allowlist lives at:

data\computer_use\approved_shortcuts\ ← one .json file per approved app

Three examples come pre-loaded: Notepad, VS Code, and Browser. You can use these as reference when adding new applications.

Adding an app

  • Create a .json file in data\computer_use\approved_shortcuts\ using the schema described in the README in that folder.
  • Ensure a matching policy entry exists in data\computer_use\app_policy.json.
  • In the Admin Panel, click Computer Use → Reload from disk to pick up the new file without restarting.

Revoking an app

Delete the app's .json file from approved_shortcuts\. The revocation takes effect immediately on the next policy check — no reload required.

Emergency Stop

The HALT Button

The Admin Panel's Computer Use tab has a red HALT button. Pressing it immediately stops all active computer-use activity and sets the mode to Off.

Use HALT if EOS does something unexpected, unsafe, or simply too autonomous for the current moment. You can re-enable and set a mode again from the Admin Panel after investigating.

!
Emergency stop
The HALT button is your emergency stop. It works immediately regardless of what EOS is doing.
Safety Layer

Confirming Actions

Some actions require your confirmation before EOS proceeds. These are actions marked soft_confirm or hard_confirm in the app policy.

When a confirmable action is about to run, a confirmation dialog appears in the Computer Use tab of the Admin Panel. EOS pauses and waits for your response — it will not proceed until you click Confirm or Deny.

Hard confirm actions require explicit acknowledgment — a soft click-through is not sufficient. The dialog will make the nature of the action clear before you respond.

Help

Troubleshooting

Symptom Fix
Computer Use not doing anything Check that the Enabled toggle is on in the Admin Panel and that the mode is not set to Off.
App not launching Check that the app has a .json file in data\computer_use\approved_shortcuts\.
Action blocked Check data\computer_use\app_policy.json and confirm the action is listed for that app.
HALT not responding Close the EOS window entirely as a last resort. This will terminate all EOS processes.
