Capabilities

Worldview

Give EOS a curated understanding of who you are — your values, reasoning style, and priorities — without waiting for it to learn everything from conversation.

Overview

What it Does

You deposit source documents — essays, notes, reflections, anything you've written — and ask EOS to extract a structured profile from them. That profile is injected into every conversation as a compact orientation signal.

EOS uses the profile for interpretive calibration: it shapes how EOS reads your intent, not what it says back to you. The profile does not change EOS's outputs directly — it changes how EOS understands the inputs you give it.

!
Interpretation, not output
The worldview profile influences interpretation, not output. EOS won't quote it back at you or reference it explicitly unless asked.
Usage

How to Use It

  1. 1
    Add source documents
    Place your source documents in data\worldview\sources\. Plain text and Markdown work best, but any readable format is accepted. These can be essays, journal entries, notes on your values, descriptions of how you think — anything that gives EOS signal about who you are.
  2. 2
    Ask EOS to run extraction
    Tell EOS: "Update the worldview profile" or "I've added new materials to worldview/sources — process them." EOS will not run extraction automatically — you must ask.
  3. 3
    EOS produces an updated profile
    EOS reads the new documents alongside the existing profile (if any) and writes an updated data\worldview\profile.md. It integrates new signal without discarding prior understanding — the profile is cumulative.
  4. 4
    Profile takes effect immediately
    The updated profile is used from the next conversation turn. No restart required.
💡
Incremental updates
You can add documents to sources\ incrementally. Run extraction again each time you add significant new material — EOS will fold it into the existing profile rather than starting over.
How It Behaves

Behavioral Rules

The worldview system follows four fixed rules:

  • Dropping a file into sources\ is a silent event — EOS will not acknowledge it unless you ask.
  • EOS never quotes the profile back at you. It uses the profile to shape interpretation, not to narrate what it knows about you.
  • Your actual expressed view in conversation always overrides the profile. What you say now takes precedence over what the profile inferred from prior writing.
  • Where the profile is uncertain, EOS treats it as uncertain — it asks rather than assumes.
Maintenance

When to Run Extraction

Run extraction whenever you add new source documents or want to refresh EOS's model of you. There is no automatic extraction — it only runs when you explicitly ask.

A reasonable cadence: run after adding significant new material (a substantial essay, a set of related notes). There is no benefit to running it after every small addition — wait until you have something meaningful to incorporate.

💡
Quality over quantity
A few well-written essays about your values and reasoning style are more useful than many short notes. Depth gives EOS better signal than volume.
The Output File

The Profile File

The extracted profile is stored at data\worldview\profile.md. EOS reads that file through the dedicated worldview_read tool rather than through workspace-only file tools. It is a plain Markdown file — you can edit it directly if you want to correct or remove anything.

EOS tracks which source files have already been processed in data\worldview\extraction_log.json. This lets it incorporate only new material on subsequent runs rather than re-reading everything.

If the profile seems wrong

  • To correct a specific section: tell EOS directly in conversation — "The profile overstates X" or "Remove the section about Y." EOS can edit the profile file on your instruction.
  • To start over: delete data\worldview\profile.md and extraction_log.json, then ask EOS to run extraction again from your source documents.
