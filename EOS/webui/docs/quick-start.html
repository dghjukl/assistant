Getting Started

Quick Start

The shortest path from zero to a running EOS system. Follow these steps in order.

ⓘ
What you'll do
Install Python → Run Setup → Verify → Start Backends → Open Browser

Estimated time: under an hour for the full install, depending on your internet connection.
Before You Begin

Prerequisites

EOS requires Python 3.11. If you already have it installed with PATH configured, skip ahead to Step 1.

  1. 1
    Download Python 3.11
    Go to python.org/downloads and download the Python 3.11 installer. During installation, check "Add Python to PATH" before clicking Install. This checkbox is on the first screen — do not miss it.
  2. 2
    Verify Python is available
    Open a Command Prompt and run: 
    python --version
    You should see Python 3.11.x. If you see an error or a different version, reinstall Python and ensure "Add Python to PATH" is checked.
⚠
Do not skip "Add Python to PATH"
If you skip "Add Python to PATH" during install, nothing will work downstream. Re-run the Python installer, choose Modify, and check that box before continuing.
Step 1

Run Setup

The setup script handles everything: pip packages, binaries, and model downloads.

  1. 1
    Right-click Setup-Full.ps1 and choose "Run with PowerShell"
    If Windows shows a security prompt about execution policy, click Open or type Y when prompted. The script will start and show progress for each component it installs.
  2. 2
    Wait for the download to complete (~13 GB)
    Setup-Full.ps1 downloads approximately 13 GB of models and binaries. This only needs to run once. Leave the window open and let it finish — do not close it mid-download.
  3. 3
    Alternative: Setup-Lite.ps1
    Setup-Lite.ps1 (~3 GB) installs supporting files only. The primary model must be added manually afterwards. Use this if you want to control which large models are downloaded, or if you are on a limited connection.
💡
Safe to re-run
Setup checks for already-installed components and skips them. If a download was interrupted or something failed, just re-run the same script — it will pick up where it left off.
Step 2

Verify

Run the verification script before starting EOS. It catches problems early so you are not debugging a running system.

python verify.py

What you should see: all checks listed as PASS in green. The script checks pip packages, llama binaries, model files, port availability, and config.json.

  • Red errors — fix these before continuing. EOS will not start correctly with unresolved errors.
  • Yellow warnings — non-fatal. Optional capabilities may be unavailable but EOS will run.
🔧
Missing model file?
If verify.py reports a missing model file, re-run Setup-Full.ps1. It will skip files that already downloaded correctly and only re-fetch what is missing or incomplete.
Step 3

Start Backends

EOS needs at least one model server running before it starts. Use the launcher for the easiest experience.

Recommended: GUI Launcher

Double-click Launch EOS.bat to open the GUI launcher. Select your desired profile and click Start. The launcher manages all backend windows for you.

Manual: Script Launchers

For the standard recommended stack, run:

start-standard.bat

This opens three console windows:

Window Port Role
Main model 8080 Primary language model — handles all chat
Tool helper 8082 Handles tool calls and function routing
Thinking helper 8083 Extended reasoning and step-by-step analysis

Wait for each window to print its ready message before continuing. The exact text varies by model, but look for a line indicating the server is listening on its port.

💡
Capability map output
When EOS starts it will print a capability map showing which services were discovered and their status. All three backends showing active means everything is ready.
Step 4

Open EOS

With backends running, start the EOS interface and open your browser.

  1. 1
    Run start-eos.bat
    Double-click start-eos.bat or run it from a terminal. Wait for the console to print Starting WebUI at http://127.0.0.1:7860/.
  2. 2
    Open the chat interface
    Navigate to:
    http://127.0.0.1:7860/
    The chat interface loads in your browser. You can start talking to EOS immediately.
  3. 3
    Optional: Open the Admin Panel
    The operator control panel is at:
    http://127.0.0.1:7860/admin
    From here you can monitor service status, adjust settings, and view diagnostics.
Optional

Optional Capabilities

None of these are required to run EOS. Enable them when you need them.

Capability How to enable
Discord Bot Add your bot token to AI personal files\Discord.txt
Google Calendar / Gmail / Drive Place your OAuth JSON credentials in AI personal files\
Computer Use Enable in the Admin Panel under Capabilities
Vision Run start-vision-gpu.bat (NVIDIA GPU required)
Voice (STT / TTS) Already active if Setup-Full.ps1 completed — no extra steps
Worldview Drop documents into data\worldview\sources\
🔧
If something fails
  • Re-run Setup-Full.ps1 — it is safe to re-run and will skip files that are already correct.
  • Re-run python verify.py — check for red errors and resolve them before restarting.
  • Run status-eos.bat to see which services are currently active, degraded, or unavailable.
  • Check the Troubleshooting page for common error messages and fixes.
