Getting Started

Quick Start

The canonical first-run path: install EOS, start the recommended backend bundle, then bootstrap EOS.

ⓘ
Default path
setup\Setup-Full.ps1 → python verify.py → launchers\start-standard.bat → start-eos.bat

Deviate only if you need a lighter install, a lower-resource stack, or manual backend control.
Before You Begin

Product vs. Entity

EOS is the platform. The entity is the runtime intelligence inside EOS. The entity's chosen name is not the product name.

Prerequisites

Install Python 3.11

EOS requires Python 3.11. If you already have it installed with PATH configured, skip ahead to Step 1.

  1. 1
    Download Python 3.11
    Go to python.org/downloads and install Python 3.11. During installation, check "Add Python to PATH".
  2. 2
    Verify Python is available
    python --version
    You should see Python 3.11.x.
Step 1

Run the Recommended Install Path

For most users, install EOS with setup\Setup-Full.ps1.

  1. 1
    Run setup\Setup-Full.ps1
    Right-click the script and choose Run with PowerShell. It installs packages, binaries, and the recommended model set.
  2. 2
    Wait for the full download to finish
    The full install is approximately 13 GB and only needs to be done once.
  3. 3
    Use Lite only when you need it
    setup\Setup-Lite.ps1 is the alternate path for manual model control or smaller initial downloads.
Step 2

Verify

Run verification before first launch.

python verify.py

Fix red errors before continuing.

Step 3

Start the Recommended Backend Bundle

For most users, the intended backend entry point is launchers\start-standard.bat.

launchers\start-standard.bat

That starts the resident baseline stack. Auxiliary thinking and creativity stay off until the executive requests them.

💡
When to use something else
start-minimal for lower resource use, start-full for the full installed resident stack, start-vision-gpu to add vision, or per-server launchers for manual control. Auxiliary reasoning servers remain policy-driven and on-demand.
Step 4

Bootstrap EOS

start-eos.bat is the canonical runtime launch path.

  1. 1
    Run start-eos.bat
    This discovers running services, builds the capability map, and starts the WebUI. It does not launch model servers itself.
  2. 2
    Open the browser UI
    http://127.0.0.1:7860/
  3. 3
    Optional: open the Admin Panel
    http://127.0.0.1:7860/admin
