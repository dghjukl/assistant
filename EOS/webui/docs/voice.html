Capabilities

Voice

EOS supports voice input (speech-to-text) and voice output (text-to-speech). The voice loop mode uses local Whisper (STT) and Piper (TTS) with no cloud dependency. The web UI uses the browser's built-in speech APIs as the STT path and Piper for TTS output (with browser TTS as fallback when Piper is unavailable).

Capabilities

Requirements

Voice requires two model files to be present on disk. Both are downloaded automatically by setup\Setup-Full.ps1.

STT Model

models\stt\ggml-small.en-q8_0.bin

TTS Models

models\tts\en_US-amy-medium.onnx
models\tts\en_US-amy-medium.onnx.json
No Configuration Needed
Voice requires no configuration beyond having these files in place. If they are present and EOS started cleanly, voice just works.
Capabilities

Voice Input (STT)

When STT is active, a microphone button appears in the chat interface. Click the button to start recording; click again to stop and insert the transcript into the message box.

The web UI uses the browser's built-in Web Speech API for transcription. In most desktop browsers (Chrome, Edge) this sends audio to the browser vendor's cloud transcription service. If local-only transcription is required, use the command-line voice loop (--voice flag), which uses a local Whisper model and never transmits audio externally.

Capabilities

Voice Output (TTS)

When the TTS model is present, EOS speaks all responses aloud. The voice is Amy (en_US, medium quality), a Piper TTS model that runs entirely on CPU.

There is no on/off toggle in the chat UI — voice output is always active when the model file is present. To disable TTS, remove or rename the .onnx file. EOS will detect its absence on next start and silently skip TTS.

Capabilities

Checking Voice Status

Run status-eos.bat from the EOS directory. The output includes a line for each voice component:

  • STT: active or STT: unavailable
  • TTS: active or TTS: unavailable

You can also check the Admin Panel → Overview tab. The STT and TTS rows appear in the server cards section with their current status badges.

Troubleshooting

Symptom Likely Cause & Fix
Mic button not appearing in chat STT model is missing or failed to load. Run status-eos.bat and check the STT line. Verify the model file is at the exact path shown above.
TTS is silent Check that your system audio is not muted and that no other application is exclusively holding the audio output device. Restart EOS after freeing the device.
Transcription is inaccurate Speak clearly and close to the microphone. The ggml-small Whisper model has known limitations with strong accents and background noise. A larger model would improve accuracy but requires more RAM.
TTS is very slow Piper TTS runs on CPU. Synthesis speed depends on your processor. On slower machines there will be a noticeable pause before speech starts. This is expected behavior, not an error.
STT Unavailable
If STT is unavailable, double-check that the model file path matches exactly:
models\stt\ggml-small.en-q8_0.bin
The filename is case-sensitive on some systems. Ensure there are no extra characters or whitespace in the filename.
