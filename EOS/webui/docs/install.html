Quick Start

Install & Setup

Everything the setup scripts do, what gets installed where, and how to recover from a failed install.

Overview

Setup Scripts

Two scripts are provided. Both are safe to re-run at any time.

Script Download size What it includes When to use
setup\Setup-Full.ps1 ~13 GB All models including primary Qwen3-8B, all binaries, TTS engine, embedding model Recommended — everything you need in one run
setup\Setup-Lite.ps1 ~3 GB Supporting files, pip packages, binaries, TTS — no primary model Use when you want to supply the primary model manually, or have a limited connection

If you use setup\Setup-Lite.ps1, you must manually place the primary model file (Qwen3-8B GGUF) into the correct models\ subdirectory before starting EOS. Run python verify.py afterwards to confirm the path is correct.

What Gets Installed

What Setup Installs

Setup runs each step in order. Every step checks whether the component is already installed before doing anything — so re-runs are fast.

  1. 1
    Check Python version
    Confirms Python 3.10 or higher is on PATH. Exits with a clear error if Python is missing or too old.
  2. 2
    Install pip packages
    Runs pip install -r requirements.txt. Installs all Python dependencies including Gradio, sentence-transformers, sounddevice, and integration libraries. Skipped if packages are already at the correct version.
  3. 3
    Download llama.cpp binaries
    Downloads both CPU and GPU builds of the llama.cpp server binary. Both are kept so EOS can select the correct one at startup based on whether a compatible GPU is detected.
  4. 4
    Download Piper TTS engine
    Downloads the Piper text-to-speech binary and the default voice model. Piper runs locally — no cloud TTS call is ever made.
  5. 5
    Download model files
    (Full only) Downloads the primary Qwen3-8B GGUF, the tool-helper model, and the thinking-helper model into the appropriate models\ subdirectories. Each file is checked for correct size before downloading — partially downloaded files are re-fetched.
  6. 6
    Verify embedding model
    Checks that the sentence-transformer embedding model is cached and usable. Downloads it via the sentence-transformers library if not already present.
Running the Script

Running Setup

Setup runs in PowerShell. No admin rights are required under normal circumstances.

  1. 1
    Right-click the .ps1 file and choose "Run with PowerShell"
    Windows may show a security dialog. Click Open or confirm with Y. The script window will open and begin printing progress.
  2. 2
    If execution policy blocks the script
    Open PowerShell as Administrator and run: 
    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
    Then close that window and right-click the script again to run it normally.
⚠
Disk space required
setup\Setup-Full.ps1 requires approximately 20 GB of free disk space on the drive where EOS is installed. Ensure you have enough space before starting. The script will fail mid-download if it runs out of space, leaving partial files that must be cleaned up before re-running.
Next Step

After Setup — Run Verify

Always run the verification script after setup completes to confirm everything is in order before starting EOS.

python verify.py

The script checks the following and prints a result for each:

Check What it tests
pip packages All required Python packages importable at correct versions
llama binaries CPU and GPU server binaries present and executable
model files All expected GGUF files present with correct file sizes
port availability Ports 8080, 8082, 8083, 7860 are free (not in use by another process)
config.json Configuration file exists, is valid JSON, and has required keys

Red errors must be resolved before starting EOS. Yellow warnings are non-fatal — optional capabilities may be limited but core operation is unaffected.

Recovery

Re-running Setup

Setup is fully idempotent. Re-running it is always safe and is the recommended recovery path for most install problems.

For every file it would download, setup checks whether that file already exists and has the expected size. If it does, the download is skipped. This means re-runs are fast — only missing or incomplete files are fetched.

💡
Interrupted download?
If a download was cut off (power loss, network drop, or you closed the window), just re-run the same setup script. It will detect the incomplete file and re-download only that file.
Troubleshooting

If Something Fails

Common setup errors and how to fix them.

Error Cause Fix
'python' is not recognized Python not on PATH Re-run the Python 3.11 installer and check "Add Python to PATH"
Execution policy error PowerShell script execution blocked Run Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser in an admin PowerShell window
Disk full / write error Less than ~20 GB free on target drive Free up disk space and re-run setup
Download failed / connection error Network interruption during model download Re-run the same setup script — it will resume incomplete downloads
verify.py reports missing model Model file absent or wrong size Re-run setup\Setup-Full.ps1 to re-fetch the missing file
