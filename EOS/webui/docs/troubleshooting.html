Troubleshooting

Common problems organized by symptom, with direct fixes.

ⓘ
Start here first
Most problems are solved by: (1) re-running Setup-Full.ps1, (2) running python verify.py and reading the output carefully, (3) checking status-eos.bat. Try these before digging into the sections below.
Troubleshooting

EOS Won't Start

Problems that prevent EOS from starting at all.

Symptom Fix
Python not recognized
Error: 'python' is not recognized as a command		 Reinstall Python 3.11 from python.org. On the first installer screen, check "Add Python to PATH" before clicking Install. Then open a new terminal and retry.
Missing module error
ModuleNotFoundError: No module named '…'		 Run pip install -r requirements.txt --no-warn-script-location from the EOS directory. If the error persists, check that you are using the correct Python environment.
config.json not found Make sure you are running start-eos.bat or python eos.py from inside the EOS directory. Do not run it from a different working directory.
Port already in use
OSError: [Errno 98] Address already in use		 Another instance of EOS or a llama-server process is already running on that port. Close all EOS windows and check Task Manager for lingering llama-server.exe or python processes, then retry.
Troubleshooting

Models Not Loading

The model server windows open but the model does not load, or a server is missing entirely.

Symptom Fix
No model file found Re-run Setup-Full.ps1. It will check each model file and re-download anything missing or incomplete.
llama-server not found Re-run Setup-Full.ps1 to reinstall the llama.cpp binaries. The binaries are placed in the bin\ folder — if that folder is missing or empty, setup did not complete.
Model loads but server doesn't respond Wait longer. Large models (7B+) can take 1–3 minutes to fully load before the server is ready to accept requests. Watch the server console window for a line confirming it is listening.
CUDA out of memory
CUDA error: out of memory		 Switch to the CPU launcher (start-*-cpu.bat) for that server, or use a smaller quantization of the model. Reduce n_gpu_layers in config.json to offload fewer layers to the GPU.
💡
Use verify.py to identify missing files
Run python verify.py before starting backends. It checks every expected model file and binary and reports exactly what is missing — saves time compared to watching server windows for error messages.
Troubleshooting

Voice Not Working

Voice input (STT) or voice output (TTS) is missing or not functioning correctly.

Symptom Fix
Microphone button missing from the UI The STT model was not found. Check that models\stt\ggml-small.en-q8_0.bin exists. If it is missing, re-run Setup-Full.ps1.
TTS is silent Check your system audio output device and volume. Verify that no other application has exclusive control of the audio output device. Restart EOS after changing audio settings.
Transcription is inaccurate Speak clearly and close to the microphone. The Whisper small model has limitations — it works well for clear speech in quiet environments but struggles with heavy accents, fast speech, or background noise.
💡
Check STT and TTS status quickly
Run status-eos.bat. It shows STT: active or STT: unavailable and the same for TTS. If either shows unavailable, the model file is missing or failed to load.

Integration Problems

Discord or Google integrations are not connecting or not responding as expected.

Symptom Fix
Discord bot offline Check the bot token in AI personal files\Discord.txt. The file should contain only the token — no extra spaces, blank lines, or quotation marks. Copy the token directly from the Discord Developer Portal.
Discord bot online but not responding to messages In the Discord Developer Portal, go to your application → Bot → Privileged Gateway Intents and enable Message Content Intent. Without this, the bot cannot read message content.
Google not connecting Delete data\google_token.json, restart EOS, then open Admin Panel → Integrations and click Re-authorize. This is the fix for expired or corrupted OAuth tokens.
Google Calendar not showing events Confirm calendar_enabled: true is set in config.json under the google block. If it is false, Calendar data is never fetched regardless of the OAuth connection state.

Memory and Context Issues

EOS is not remembering things, not applying the worldview profile, or not seeing workspace files.

Symptom Fix
EOS doesn't remember past conversations Check that data\memory_store\ exists and is not empty. If it is missing, ChromaDB was not initialized — re-run python verify.py and check for memory-related errors. The Memory tab in the admin panel shows stored memory count.
Worldview profile not applying Verify that data\worldview\profile.md exists and contains content. If it is empty or missing, the worldview extraction has not run. Add source documents to data\worldview\sources\ and trigger an extraction.
Workspace files not seen by EOS Files must be placed in data\workspace\context\ to be automatically injected into context. Files placed in the workspace root or other subdirectories are not automatically visible — move them into the context\ subfolder.

Performance Issues

EOS is slow, uses excessive memory, or hangs during certain operations.

Symptom Fix
Chat is very slow (many seconds per token) The main model is running on CPU. GPU inference is strongly recommended for acceptable speed. Use a start-*-gpu.bat launcher and confirm the server console shows GPU layers loaded. Check that CUDA drivers are up to date.
TTS is very slow Piper TTS runs on CPU — this is expected behavior. TTS speed scales with the length of the response. There is no GPU-accelerated path for Piper in the current build.
High memory usage Reduce the number of simultaneously running backends. Each loaded model occupies RAM (and VRAM on GPU). Stop backends you are not actively using and restart them when needed.
EOS hangs on tool calls Check that the tool helper server is running on port 8082. Run status-eos.bat — if Tool helper shows unavailable, start it with start-tools-gpu.bat or start-tools-cpu.bat.

When All Else Fails

A reliable reset sequence that resolves the majority of persistent problems.

  1. 1
    Close all EOS windows
    Close every EOS console and browser tab. Check Task Manager for any remaining llama-server.exe or python processes related to EOS and end them before continuing.
  2. 2
    Run python verify.py and fix all red errors
    python verify.py
    Read the output carefully. Red errors must be resolved before EOS will start correctly. Yellow warnings are non-fatal and can be addressed later.
  3. 3
    Re-run Setup-Full.ps1
    Right-click Setup-Full.ps1 and choose Run with PowerShell. It is safe to re-run — it checks each component and skips anything already correctly installed. Only missing or broken components are re-downloaded or reinstalled.
  4. 4
    Start minimal first, then add optional backends
    Run start-minimal.bat to bring up only the core backends (main model + tool helper), then run start-eos.bat. Confirm the core system works in the browser before launching optional backends like thinking, vision, or creativity. This isolates whether a problem is in the core or in an optional component.
