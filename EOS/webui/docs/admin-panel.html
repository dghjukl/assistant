Admin Panel

The operator's control surface for monitoring, tuning, and managing EOS at runtime.

Live Changes & Persistence
All changes made in the admin panel are live immediately. Most reset to config.json values on restart. To make a change permanent, edit config.json.
Reference

Accessing the Admin Panel

The admin panel is available at:

http://127.0.0.1:7860/admin

You can also reach it from the chat interface top bar — look for the Admin link in the upper-right corner of the EOS chat UI. The panel opens in the same browser tab.

Tab Overview

The admin panel is organized into twelve tabs. Each covers a distinct part of the system.

Tab What It Shows
Overview Server health dashboard, session ID, turn count, and inference server status for all running services.
Tools All loaded tools with the ability to enable or disable individual tools at runtime.
Control & Permissions Capability toggles, runtime permission classes, toolpack management, and the permission audit log.
Tool Registry Diagnostics Raw tool registry state and shadow database state for low-level debugging.
Live Logs Real-time system log stream, color-coded by level.
Main Model Per-server inference log for the main language model, with hardware badge (GPU/CPU).
Tool Svr. Per-server inference log for the tool-calling model, with hardware badge.
Memory Vector memory statistics, retrieval log, and memory event history.
Cognition Turn traces, memory influence events, reflection events, and internal state changes.
Computer Use Mode switching controls, the HALT button, application allowlist, and pending confirmation queue.
Integrations Google Workspace connection status and live data for all external integrations.
Config Read-only view of the currently loaded config.json.
Export Generate a full diagnostic bundle for bug reports or persistent-issue diagnosis.

Overview Tab

The Overview tab is the server health dashboard. It shows each inference server with its current status, hardware, model, and port.

Inference Server Cards

Each server appears as a card showing:

  • Status badge: active, degraded, or unavailable
  • Hardware badge indicating GPU or CPU inference
  • Model name currently loaded on that server
  • Port number the server is listening on

Session Info

Below the server cards, the session block shows:

  • Entity name — EOS's configured identity name
  • Session ID — unique identifier for the current runtime session
  • Turn count — number of conversation turns completed this session

When Something Seems Wrong

The Overview tab should be your first stop when behavior is unexpected. Check whether all servers show active. A degraded or unavailable status means the corresponding server did not load correctly — the model may be missing, the port may be in use, or hardware resources may be insufficient. The server cards show enough context to identify which server failed and on which hardware.

Control & Permissions Tab

This tab is the primary control surface for adjusting what EOS is allowed to do at runtime. Changes take effect immediately.

Capabilities

Live toggles for all capability groups — Perception, Cognition, Action, and Initiative. Toggling a group off removes that class of behavior immediately. See the Capabilities & Autonomy page for what each group controls.

Runtime Permission Classes

Allow or block tool permission classes immediately. These are the security controls that gate whether EOS can use tools in sensitive categories such as file system access, network calls, or shell execution. Blocking a class here takes effect without a restart and does not require editing config.json.

Toolpack Management

Enable or disable individual tools or entire toolpacks at runtime. Use this to temporarily pull a tool out of rotation without restarting EOS. Disabled tools remain loaded in memory but will not be offered to or invoked by the model.

Audit Log

The bottom of the tab shows a timestamped audit log of recent permission changes — who changed what and when. Useful for reviewing what was adjusted during a session.

Live Logs Tab

The Live Logs tab streams the system log in real time, color-coded by severity level. It is the fastest way to see what EOS is doing moment-to-moment.

The log stream includes:

  • Tool invocations and their results
  • Memory retrieval events
  • External service calls (Google, Discord, etc.)
  • Errors and warnings with stack context
Debugging Tip
Keep the Live Logs tab open in a second browser tab while testing integrations. You can watch what EOS is doing in real time without interrupting the chat session in your first tab.
Reference

Export Tab

The Export tab generates a full diagnostic bundle as a JSON file. Use this when filing a bug report or diagnosing a persistent issue that is hard to reproduce.

What the Bundle Contains

  • Current runtime state snapshot
  • Status of all inference servers
  • Recent log entries from the live log stream
  • A snapshot of the loaded config.json

Where It Saves

The bundle is saved to the exports/ folder inside your EOS directory and is simultaneously offered as a browser download. Both copies are identical — keep whichever is convenient.
