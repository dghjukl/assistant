Capabilities

Access Tiers & LAN Access

EOS classifies every incoming request by its network origin and enforces a distinct policy for each tier — localhost, LAN, and external. This lets you safely connect phones, tablets, and other LAN devices without giving them the same unlimited trust as the machine running EOS.

ⓘ
Default posture
Out of the box: localhost is fully open (no auth, no rate limit), LAN is enabled but requires a session token, and external is fully disabled. You can change any of this from the Access Tiers tab in the admin panel — no restart required.
How It Works

The Three Tiers

Every request is tagged with its origin tier before any other processing occurs. The tier determines what the caller can do.

TierAddressesDefault policy
localhost 127.0.0.0/8, ::1 Fully open — no auth required, no rate limit, all paths accessible.
lan 10.x, 172.16–31.x, 192.168.x, 169.254.x, fc00::/7, fe80::/10 Chat & admin accessible, requires LAN session token, 60 req/min limit.
external All other routable addresses Fully disabled — all requests are rejected with 403.

Tier detection uses the direct TCP connection address. If EOS is behind a reverse proxy on the same machine you can set EOS_TRUST_PROXY=1 to read X-Real-IP / X-Forwarded-For instead.

Configuration

Policy Fields

Each tier has these configurable fields. Defaults come from config.json; runtime changes are written to data/access_tiers.json and survive restarts.

FieldTypeDescription
enabledboolIf false, every request from this tier is rejected with 403.
chat_enabledboolWhether /api/chat and other public API routes are accessible.
admin_enabledboolWhether /admin/* paths can be reached (admin token still required on top).
require_authboolWhether a LAN session token is required for non-admin, non-exempt routes.
rate_limit_rpmintRequests per minute per client IP (0 = unlimited).
rate_limit_burstintExtra requests allowed above the steady rate before the limit kicks in.
session_ttl_secintHow long a LAN session token remains valid (minimum 60 seconds).
Step-by-Step

Connecting a LAN Device

Follow these steps to pair a phone, tablet, or second computer on your local network.

  1. 1
    Open the Access Tiers tab
    In the admin panel, click the Access Tiers tab. Confirm the lan tier shows ENABLED. If it shows DISABLED, click Edit and toggle it on.
  2. 2
    Generate a pairing code
    Click Generate Pairing Code. A 22-character random code appears — it is valid for 5 minutes and can only be used once. Keep this browser tab open.
  3. 3
    Send the pairing request from the device
    On the LAN device, make a POST request to EOS: 
    POST http://<EOS-IP>:7860/api/auth/lan/pair
Content-Type: application/json

{"code": "<paste-code-here>", "label": "My Phone"}
    EOS returns a session token: 
    {"ok": true, "data": {"token": "aBcD…", "expires_at": 1740000000, "tier": "lan"}}
    Save the token value — you will send it with every subsequent request.
  4. 4
    Include the token in API requests
    Add the header X-Lan-Token: <token> to every request: 
    POST http://<EOS-IP>:7860/api/chat
Content-Type: application/json
X-Lan-Token: aBcD…

{"message": "Hello from my phone!"}
    Without this header, LAN requests to the chat API will receive a 401 response.
  5. 5
    Verify the session
    Check your session is recognised: 
    GET http://<EOS-IP>:7860/api/auth/lan/status
X-Lan-Token: aBcD…
    Returns {"origin_tier": "lan", "session_valid": true, …}. The session is also listed in the Active LAN Sessions section of the Access Tiers tab.
Operator Controls

Managing Tiers from the Admin Panel

Viewing tier status

The Access Tiers tab shows three cards — one per tier — with every policy field and a colour-coded enabled/disabled badge. Click ↻ Refresh to reload live data.

Editing a tier

Click Edit on any tier card (or the tier button in the Edit Tier Policy section). A form pre-fills with current values. Change what you need and click Save Changes. Changes take effect immediately — no restart required.

Revoking a session

The Active LAN Sessions table lists every paired device (IP, label, expiry). Click Revoke next to any session to immediately invalidate that token. The device will need to re-pair.

Disabling LAN access entirely

Click Edit on the lan tier, uncheck Tier enabled, and save. All LAN requests will receive 403 Forbidden until you re-enable the tier.

Observability

Audit Visibility

Every admin action and tool execution now records origin_tier and client_ip in the audit log. You can see these fields in the Audit tab of the admin panel. This makes it immediately visible whether a sensitive action was taken locally or from a LAN device.

The Live Logs tab also shows origin classification events (pairing code generated, session created, rate limit triggered).

Protection

Rate Limiting

Rate limiting is enforced per client IP using a 60-second sliding window. The limit is applied independently per tier, before any authentication check.

TierDefault limitDefault burst
localhostunlimited—
lan60 req/min15
external20 req/min5

When a client exceeds the limit, EOS responds with 429 Too Many Requests and a Retry-After: 60 header. Adjust limits in the Edit Tier Policy form or directly in config.json under access_tiers.<tier>.rate_limit_rpm.

Reference

Configuration Reference

The access_tiers block in config.json sets the baseline for each tier. Runtime overrides written from the admin panel are stored in data/access_tiers.json (runtime file wins on startup).

"access_tiers": {
  "localhost": {
    "enabled": true,
    "chat_enabled": true,
    "admin_enabled": true,
    "require_auth": false,
    "rate_limit_rpm": 0,
    "rate_limit_burst": 0,
    "session_ttl_sec": 86400
  },
  "lan": {
    "enabled": true,
    "chat_enabled": true,
    "admin_enabled": true,
    "require_auth": true,
    "rate_limit_rpm": 60,
    "rate_limit_burst": 15,
    "session_ttl_sec": 86400
  },
  "external": {
    "enabled": false,
    "chat_enabled": false,
    "admin_enabled": false,
    "require_auth": true,
    "rate_limit_rpm": 20,
    "rate_limit_burst": 5,
    "session_ttl_sec": 3600
  }
}
